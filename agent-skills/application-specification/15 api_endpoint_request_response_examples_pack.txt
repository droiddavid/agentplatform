API ENDPOINT REQUEST AND RESPONSE EXAMPLE PACK
Runtime AI Agent Platform v1

1. PURPOSE
This pack provides canonical request and response examples for the API surface defined in the master specification. These examples are implementation targets. Final field names may expand, but the semantics must remain aligned.

2. CONVENTIONS
- Base path: /api
- JSON request and response bodies unless stated otherwise.
- Auth via Bearer token for protected routes.
- Timestamps use ISO-8601 UTC strings.
- Errors return a standard envelope.

3. STANDARD SUCCESS ENVELOPE
{
  "success": true,
  "data": {}
}

4. STANDARD ERROR ENVELOPE
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "One or more fields are invalid.",
    "details": [
      { "field": "email", "message": "Email is required." }
    ]
  }
}

19. API ACCEPTANCE SUMMARY
This pack is accepted only if:
- Every core frontend screen has enough API support.
- All approval workflows are represented.
- Run, task, agent, billing, privacy, and connection examples are sufficient to scaffold DTOs and controllers.
- The examples remain aligned with the master specification and UX pack.
