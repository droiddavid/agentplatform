ACCEPTANCE TEST PACK
Runtime AI Agent Platform
Version 1

==================================================
1. PURPOSE
==================================================

This pack defines the product-level and module-level acceptance tests for the
consumer-ready runtime AI agent platform. It is designed for:
- implementation validation
- QA planning
- automated E2E planning
- release readiness

Tests are written in plain language first so a coding agent can convert them
into automated test suites later.

==================================================
2. GLOBAL ACCEPTANCE CRITERIA
==================================================

The application is release-ready only if all of the following are true:

1. A non-technical user can create an account and complete onboarding.
2. A user can create an agent through wizard, natural language, and templates.
3. A user can start a task and see a run begin.
4. Two or more agents can communicate within a run.
5. External actions are blocked for approval by default.
6. Remembered approval rules work correctly when enabled.
7. Documents can be created and viewed.
8. Connections can be added, tested, and revoked.
9. Free tier limits are enforced correctly.
10. Privacy controls for export and deletion work.
11. Run logs and event streams remain consistent with persisted data.
12. The UI works on desktop and mobile-responsive layouts.

==================================================
3. TEST DATA PRECONDITIONS
==================================================

Create or seed:
- one FREE user
- one LOCAL_PRO user
- one CLOUD_PRO user
- seeded plans and entitlements
- seeded templates
- seeded tool definitions
- at least one simulated/test connector for each connection type

==================================================
17. RELEASE READINESS CHECKLIST
==================================================

Release candidate is accepted only if:
1. All critical path tests pass.
2. No Sev-1 defects remain.
3. Billing gates are correct.
4. Approval system cannot be bypassed accidentally.
5. User-facing errors are understandable.
6. PWA shell loads correctly.
7. Core seeded templates render properly.
8. Database migrations run from empty schema without failure.
