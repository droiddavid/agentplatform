PROJECT BOOTSTRAP PACK
Runtime AI Agent Platform
Version 1

==================================================
1. PURPOSE
==================================================

This pack provides the exact bootstrap sequence for starting the project from zero.
It defines:
- repository layout
- initial folders
- starter files
- first commands
- first implementation order
- first-run verification sequence

This is designed so a coding agent or developer can begin building immediately
without inventing structure.

==================================================
2. REPOSITORY STRATEGY
==================================================

Recommended top-level repository:

agentflow-platform/
  frontend/
  backend/
  docs/
  scripts/
  .gitignore
  README.md

Purpose:
- frontend contains Angular application
- backend contains Spring Boot application
- docs may store the specification packs
- scripts may store local bootstrap helpers

==================================================
3. TOP-LEVEL STARTER FILES
==================================================

Create:
- README.md
- .gitignore
- docs/README.md
- scripts/README.md

==================================================
7. FIRST COMMAND ORDER
==================================================

7.1 Top-level

1. create repository root
2. initialize git
3. create frontend and backend directories
4. copy specification packs into docs if desired

7.2 Frontend startup command sequence example

1. create Angular app in frontend
2. install Tailwind
3. add routing setup
4. create folder structure
5. add core app shell files
6. wire minimal routes for landing, sign-in, sign-up
7. verify frontend compiles

7.3 Backend startup command sequence example

1. create Spring Boot app in backend
2. add dependencies
3. create package structure
4. add application config files
5. configure datasource and Flyway
6. add first migration
7. verify backend starts
8. verify migration applies to empty database

==================================================
8. FIRST IMPLEMENTATION MILESTONE
==================================================

Milestone 1 goal:
A user can open the app, see landing page, sign up, sign in, and the backend
is connected to MySQL with Flyway migrations working.

This milestone includes:
- frontend route skeleton
- backend auth skeleton
- users table
- config loading
- basic CORS
- JWT placeholders
- local environment wiring

==================================================
9. SECOND IMPLEMENTATION MILESTONE
==================================================

Milestone 2 goal:
User onboarding works and template browsing is visible.

This milestone includes:
- onboarding pages
- onboarding backend
- template category and template migrations
- seeded template data
- template listing APIs
- frontend template browser

==================================================
10. THIRD IMPLEMENTATION MILESTONE
==================================================

Milestone 3 goal:
User can create an agent and see it in the agent list.

This milestone includes:
- agent migrations
- agent entities and DTOs
- agent create/edit APIs
- wizard flow
- template-based instantiate flow
- basic natural language parsing flow

==================================================
11. FIRST-RUN VERIFICATION CHECKLIST
==================================================

After bootstrap, verify:
1. frontend starts locally
2. backend starts locally
3. database connection works
4. Flyway migration V1 runs
5. health endpoint responds
6. landing page renders
7. sign-in route loads
8. sign-up route loads

==================================================
13. BOOTSTRAP DECISIONS TO KEEP CONSISTENT
==================================================

1. Use runtime-defined agents from day one.
2. Do not build a fake static agent system as a temporary shortcut.
3. Keep DB-backed run events in mind before building workspace UI.
4. Keep approval-first architecture in place from the beginning.
5. Design routes and backend packages so later modules fit cleanly.

==================================================
14. FINAL BOOTSTRAP DIRECTIVE
==================================================

The project is bootstrapped correctly only when:
- directory structures align with spec packs
- base apps compile and start
- migrations run
- auth foundation is in place
- the implementation path remains aligned with the full specification
