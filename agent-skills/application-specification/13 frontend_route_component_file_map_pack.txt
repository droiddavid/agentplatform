FRONTEND ROUTE, COMPONENT, AND FILE MAP PACK
Runtime AI Agent Platform
Version 1
Frontend Stack: Angular, TypeScript, Signals, Tailwind, PWA

==================================================
1. PURPOSE
==================================================

This pack defines the frontend application structure in prescriptive detail:
- route map
- feature folder map
- page components
- shared components
- core services
- models
- guards
- state services
- file naming rules

The goal is to make implementation deterministic for a coding agent.

==================================================
12. ROUTING IMPLEMENTATION RULES
==================================================

1. Use lazy loading for every feature area.
2. Group public and authenticated shells clearly.
3. Do not place heavy workspace modules in the initial bundle.
4. Use route resolvers only where truly useful, otherwise load in component store.
5. Route params must be strongly typed in helper utilities when possible.

==================================================
17. NAMING RULES
==================================================

1. Page components end with PageComponent.
2. Presentational reusable pieces end with Component.
3. Service wrappers for API calls end with .api.ts or Service when cross-cutting.
4. Feature state files end with .store.ts.
5. Models grouped by feature in .models.ts.
6. Avoid one giant shared models file.
