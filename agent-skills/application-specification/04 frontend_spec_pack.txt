FRONTEND SPECIFICATION PACK
Project: Consumer Runtime AI Agent Platform
Version: V1
Frontend Stack: Angular, TypeScript, Signals, Tailwind, PWA
Purpose: This document defines the complete frontend implementation specification for a consumer facing runtime AI agent platform. It is written so an implementation agent can build the frontend without further clarification.

1. FRONTEND MISSION
The frontend must allow a non technical user to understand, create, supervise, approve, and benefit from AI agents that can do work, communicate with one another, and take approved actions. The interface must remove technical complexity and expose only consumer understandable concepts.

2. FRONTEND PRINCIPLES
2.1 Agents are runtime entities. The frontend must never assume compile time agent classes.
2.2 The UI must explain what is happening in plain language.
2.3 Every important state transition must be visible.
2.4 Approval is central. External actions are blocked by default and surfaced clearly.
2.5 Desktop and mobile responsive usability are both required.
2.6 Version 1 supports a web application with mobile responsive PWA behavior.
2.7 The architecture must allow native mobile later.

7. STATE MANAGEMENT
7.1 Global state
Must be limited to:
- auth session
- current user summary
- notification list
- PWA status
- active usage summary
- feature entitlement summary

7.2 Feature local stores
Each major route domain must have its own local signal based store:
- AgentsStore
- TasksStore
- RunsStore
- WorkspaceStore
- ApprovalsStore
- ConnectionsStore
- DocumentsStore
- BillingStore
- PrivacyStore

7.3 Realtime state
Realtime updates must merge streamed run events into local state without requiring full page reload.

17. ACCEPTANCE CRITERIA
Frontend is accepted only if:
- a non technical user can sign up and onboard
- a user can create an agent using wizard, text, or template
- two agents can be seen collaborating in the workspace
- approvals are understandable and actionable
- graph view and task board function on desktop and mobile
- memory and privacy screens are usable
- billing and free limit information are clear
- the app is installable as a PWA
