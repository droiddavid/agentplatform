BACKEND PACKAGE, CLASS, AND FILE MAP PACK
Runtime AI Agent Platform
Version 1
Backend Stack: Spring Boot, Java, MySQL

==================================================
1. PURPOSE
==================================================

This pack defines the backend package structure, class responsibilities,
file map, and implementation ordering for a production ready consumer AI
agent platform. It is intentionally prescriptive so that a coding agent
can build the backend with minimal ambiguity.

==================================================
3. PACKAGE RULES
==================================================

1. Each domain package owns its controller, service, DTOs, entities, repository, mapper, and enums.
2. Shared abstractions live in common or config, not in arbitrary feature folders.
3. Domain services must not directly depend on unrelated domain repositories unless there is a clear cross-domain orchestration need.
4. Cross-domain workflows should go through orchestration services.
5. Controllers expose API boundary DTOs, not entities.
6. Repositories return entities only within the service layer.
7. All externally visible IDs should be ULID style strings.

==================================================
27. MAPPER RULES
==================================================

1. Every package should contain a mapper class.
2. Mappers translate between entities and DTOs.
3. Business logic must not live in mappers.
4. Never expose encrypted secrets or internal token hashes in response DTOs.

==================================================
28. REPOSITORY RULES
==================================================

1. Use Spring Data repositories.
2. Keep custom queries in repository interfaces or dedicated query repositories.
3. Avoid placing heavy business branching in repositories.
4. Prefer specification or query objects for complex filtering if needed.

==================================================
29. EXCEPTION HANDLING FILES
==================================================

Create:
- GlobalExceptionHandler.java
- ErrorCode.java
- ValidationErrorItem.java

Behavior:
- map domain exceptions to consistent HTTP responses
- produce machine-readable and user-friendly error payloads
