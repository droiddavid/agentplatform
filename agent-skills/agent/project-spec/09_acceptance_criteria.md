# Acceptance Criteria

The system is acceptable only if all of the following are true.

## Core Product

- user can sign up and sign in
- onboarding works
- agent creation works by wizard
- agent creation works by natural language
- agent creation works from templates
- user can create a task
- user can start a run

## Collaboration

- two or more agents can collaborate in a run
- direct peer messaging works
- shared task board works
- graph view shows meaningful relationships
- critique and delegation flows work
- spawn proposals work

## Approval System

- all external actions require approval by default
- user can approve once
- user can approve for task
- user can approve for agent
- user can approve for user
- user can deny
- ask-every-time behavior works
- approval logic cannot be bypassed accidentally

## Tools and Outputs

- supported tools can execute under correct policy
- documents can be created and viewed
- document export works
- connections can be added, tested, and revoked

## Billing and Privacy

- free plan enforces five total agent uses
- Local Pro unlocks local provider access
- Cloud Pro unlocks cloud provider access
- memory can be viewed
- memory can be deleted
- data export works
- account deletion flow works

## Realtime and UX

- run events stream correctly
- notifications stream correctly
- desktop UX works
- mobile-responsive UX works
- key flows remain understandable to non-technical users

## Release Rule

Do not mark the system complete until these criteria are verified through tests,
manual checks, or both.
