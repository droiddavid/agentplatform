# Frontend Structure

## Frontend Stack

- Angular
- TypeScript
- Angular signals
- Tailwind
- Angular Router
- PWA support
- graph visualization support
- SSE or WebSocket client support

## Feature Areas

- landing
- auth
- onboarding
- dashboard
- agents
- tasks
- runs
- workspace
- approvals
- connections
- documents
- billing
- privacy
- settings
- help

## Frontend Rules

- use lazy-loaded feature routes
- use signals and local stores
- keep user-facing wording plain-language
- make approvals highly visible
- support desktop, tablet, and mobile widths
- implement loading, empty, and error states

## Critical Workspace Features

The workspace must support:

- agent roster
- chat thread or event activity
- shared task board
- graph view
- context panel
- approval review surface

## PWA Rules

Frontend must support:

- manifest
- service worker lifecycle
- installability behavior where applicable
- graceful offline UX
